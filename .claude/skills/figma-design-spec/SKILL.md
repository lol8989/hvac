# 도면검도시스템 — 화면설계서 작성 지침

> 이 지침은 Figma 화면설계서 산출물에만 적용된다. `POC/` React 앱 구현에는 적용되지 않는다(POC는 `POC/CLAUDE.md` + LG 디자인시스템을 따른다).

## 역할 분담 (핵심 원칙)

- **사용자**: 양식 구조 정의, 화면명/요구사항ID/화면ID/Description 제공, UI 직접 배치
- **AI**: 불렛포인트 작성, UI 유형 컴포넌트 표기, 화면 명세 프레임/컴포넌트 테이블 프레임/주석 생성

---

## 작업 워크플로우

1. **사용자**: Figma 페이지 내 프레임 복제 (양식 틀 준비)
2. **사용자**: WBS, 기능명세서, 캡처 사진 등 참고자료 첨부
3. **AI (선행)**: 참고자료 기반으로 내용 작성 — 불렛포인트, UI 유형 컴포넌트 플레이스홀더, 화면 명세 프레임, 컴포넌트 테이블 프레임, 주석 번호 배치
4. **사용자 (후행)**: AI가 작성한 프레임 위에 Figma 컴포넌트 라이브러리로 실제 UI 배치
5. **반복**: 상세 화면, (공통), (예외) 프레임도 동일한 순서로 진행

참고자료 활용 기준:
- WBS → 불렛포인트 도출
- 기능명세서 → 불렛포인트 상세화 + 상태 표기
- 캡처 사진 → UI 플레이스홀더 위치/크기 참고

---

## 프로젝트 개요

본 프로젝트는 도면을 확인하는 서비스를 구축하는 프로젝트이다. Figma, draw.io, Balsamiq과 같이 도면 콘텐츠를 전체 화면 내에서 최대한 넓게 확인할 수 있도록 설계한다.

---

## UI/UX 원칙

화면설계서 작성 시 UI/UX를 사용자에게 가장 편안한 형태로 작성한다.

### 도면 뷰어 중심 레이아웃

- **콘텐츠 영역 최대화**: 도면 뷰어/캔버스 영역이 화면의 주(主) 영역을 차지하도록 설계한다
- **사이드 패널 접기/펼치기**: 메뉴, 속성, 레이어 목록 등 부가 기능은 사이드 패널에 배치하고, 접기(collapse)/펼치기(expand) 토글로 필요할 때만 노출한다
- **사이드 패널 기본 상태**: 기본 화면(요약)에서는 사이드 패널을 접힌 상태로 표시하고, 상세 프레임에서 펼친 상태를 별도로 표현한다
- **토글 버튼 위치**: 사이드 패널 경계에 화살표(◀/▶) 또는 햄버거 아이콘 토글 버튼을 배치한다
- **참고 서비스 패턴**: Figma(좌측 레이어 패널 + 우측 속성 패널), draw.io(좌측 도형 패널), Balsamiq(좌측 컴포넌트 패널) 형태를 참고한다

---

## 화면 표기 규칙

- **불렛포인트**: 각 화면의 기능/동작을 불렛포인트로 작성
- **UI 컴포넌트 표기**: 어떤 UI인지 컴포넌트로 표기 (버튼, 입력필드, 드롭다운, 체크박스 등)
- **(공통)**: 공통화면에 해당하는 경우 화면명에 (공통) 표기
- **(예외)**: 예외처리 화면의 경우 (예외) 표기 → 각 예외 케이스별 별도 프레임 생성
- **드롭다운**: 요약 화면 → 상세 화면 순서로 구성. 상세 화면에서 드롭다운이 펼쳐진 리스트 상태를 별도 프레임으로 생성

---

## UI 플레이스홀더 규칙

- Dashed Stroke 사각형 + 라벨로 UI 위치/유형 표시: `[Button]`, `[Input]`, `[Dropdown]`, `[Table]`, `[Modal]`, `[Toggle]`, `[Checkbox]`, `[Radio]`, `[Tab]`, `[Search]`, `[Calendar]`, `[Slider]`, `[Tooltip]`, `[Toast]`, `[Card]`, `[List]`
- 플레이스홀더 크기는 실제 UI 예상 크기에 맞춰 배치
- 사용자가 Figma 컴포넌트 라이브러리로 교체할 수 있도록 그룹화하지 않고 개별 레이어로 배치

---

## 프레임 구성 순서 (1화면 = N프레임)

1. **기본 상태 (요약)** — 초기 로딩 시 보이는 기본 화면
2. **상세 상태** — 인터랙션 후 펼쳐진/확장된 화면
3. **(공통)** — 여러 화면에서 재사용되는 공통 컴포넌트 화면
4. **(예외)** — 예외/에러 케이스별 별도 화면

---

## 상세 화면 프레임 작성 원칙

- **복제 기반**: 직전 기본 화면 프레임을 복제하여 상세 프레임 작성
- **주석 넘버링**: 이미 앞 프레임에서 설명된 컴포넌트의 주석 원형은 제거, 새로 추가된 컴포넌트에만 배치 (번호는 이전 프레임 번호 연속)
- **오버레이 표현**: 패널/모달/드롭다운 리스트 등 오버레이는 기존 UI 위에 겹쳐 표현. 공간이 필요한 경우 배경 영역(채팅 영역 등) 너비를 축소하여 확보
- **화면명 규칙**: `[원래 화면명] (상세 - [변경 내용])` 형식으로 헤더 테이블에 기재
- **불렛포인트**: 해당 상세 프레임에서 추가/변경된 동작만 기술 (앞 프레임과 동일한 항목은 생략 또는 "N페이지와 동일" 표기)

---

## 불렛포인트 작성 포맷

- 동그라미 숫자 기호(①②③...) 사용 금지 → 일반 숫자(1, 2, 3...)로 표기
- 형식: `[동작주체] + [동작] + [대상] + [조건/결과]`
- 예시: "사용자가 검색 버튼을 클릭하면 → 검색 결과 리스트가 하단에 표시된다"
- 주석 번호(①②③)와 불렛포인트 1:1 매핑
- 한 불렛 = 하나의 기능/동작만 기술

---

## 주석 넘버링 순서

- 주석 원형(①②③...)은 좌→우, 위→아래 순서로 배치

---

## 상태 표기

| 상태 | 설명 |
|------|------|
| [기본] | 초기 상태 |
| [활성] | 선택/포커스/호버 |
| [비활성] | disabled |
| [에러] | 유효성 검증 실패 |
| [로딩] | 스피너/스켈레톤 |
| [완료] | 작업 성공 |
| [빈값] | empty state |

## 인터랙션 상세 규칙

### 네비게이션
- 페이지 이동 경로 화살표/주석 명시, 탭별 별도 프레임, 브레드크럼 표시

### 입력/폼
- placeholder, 유효성 규칙, 에러 메시지, 필수(*) 표기, 자동완성/날짜선택 → 별도 프레임

### 팝업/모달
- 배경 딤 처리 + 별도 프레임, 토스트 위치/자동 사라짐 시간 명시

### 리스트/테이블
- 정렬(▲▼), 필터 전/후, 페이지네이션, 단일/다중선택 구분, 빈 상태 안내

### 드래그&드롭
- 시작/이동 중/드롭 완료 각 상태 표기

### 반응형/조건부
- 권한별(관리자/일반) 별도 프레임, 조건부 노출 UI 명시

---

## 데이터 바인딩

- 테이블 컬럼명, 데이터타입(텍스트/숫자/날짜/boolean), 정렬 기본값 명시
- 기본값/placeholder 정의, 입력필드별 maxlength 명시
- API 연동 포인트, 요청/응답 데이터 항목 주석 표기
- 데이터 갱신 주기: 실시간/폴링/수동 새로고침 구분

---

## 화면 흐름도 (Screen Flow)

- 화면 간 이동 관계를 별도 흐름도 프레임으로 생성
- 화살표 + 트리거 조건 명시, 뒤로가기 경로, 딥링크 표시

---

## 파일 처리

- **업로드**: 허용 형식, 최대 용량, 다중 업로드 여부, 프로그레스 바, 취소 가능 여부
- **다운로드**: 파일명 규칙, 일괄 다운로드(ZIP) 지원 여부
- **미리보기**: 지원 파일 형식 명시, 드래그앤드롭 드롭존 표시

---

## 검색 로직

- 조건 조합 AND/OR 규칙, 초기화 버튼, 결과 0건 빈 상태 안내, 최근 검색어 여부
- 실시간 검색 vs 검색 버튼 클릭 방식 구분

---

## 이탈 방지 (Unsaved Changes)

- 뒤로가기/창 닫기 시 확인 팝업: "작성 중인 내용이 삭제됩니다"
- 자동 임시저장 여부 및 저장 주기 명시

---

## 알림/배지

- 읽지 않은 알림 개수 표시 위치, 읽음/안읽음 상태 구분
- 알림 클릭 시 딥링크 이동, 알림 유형 구분(시스템/사용자/긴급)

---

## 접근성/키보드

- Tab 순서: 좌→우, 상→하
- Enter: 폼 제출 / Esc: 모달 닫기
- 포커스 시각적 표시, 주요 UI에 aria-label 명시

---

## 세션/인증

- 세션 만료 팝업 → 로그인 리다이렉트, 자동 로그아웃 조건 정의
- 권한 없는 접근: 403 또는 안내 팝업, 동시 로그인 처리 방식, 토큰 자동 갱신 여부

---

## 애니메이션/트랜지션

- 화면 전환 효과 유형, 아코디언 열림/닫힘 속도(ms)
- 토스트 슬라이드인 + 자동 사라짐 시간(예: 3초), 스피너 vs 스켈레톤 기준

---

## 버전/이력

- 수정 이력 테이블(누가/언제/무엇), 버전 비교 diff 화면, 복원 시 확인 팝업

---

## 마스터 요소

- GNB 컴포넌트를 Figma 컴포넌트로 관리
- 헤더테이블(화면명/화면ID/요구사항ID/Description)은 와이어프레임 프레임 내부에 넣지 않는다 → 화면 명세 프레임의 메타 필드(화면명/화면ID/화면경로)로 대체
- 헤더 폰트: Noto Sans KR Medium / 7pt

## 화면 명세 & 컴포넌트 테이블 (2-프레임 체계)

> 핵심 원칙: 와이어프레임 프레임 **내부에 명세를 넣지 않고**, 와이어프레임 프레임 **오른쪽에 별도 프레임 2개**를 배치한다.

### 프레임 배치 순서 (좌→우)

1. **와이어프레임 프레임** (1440 x 가변) — 화면 UI 와이어프레임 + 주석 원형
2. **화면 명세 프레임** (435 x 가변) — 화면 메타정보 + 기능 상세 불렛포인트
3. **컴포넌트 테이블 프레임** (752 x 가변) — 주석 번호별 컴포넌트명 + 설명

### 화면 명세 프레임 (435 x 가변)

| 요소 | 스타일 |
|------|--------|
| **프레임** | Fill #FFFFFF, Stroke #DDDDDD (0.5pt), cornerRadius 0 |
| **타이틀 바** | Rectangle h=36, Fill #262626 (상단) |
| **타이틀 텍스트** | "화면 명세", Noto Sans KR Bold 12pt, #FFFFFF, x=16 y=9 |
| **메타 라벨** | "화면명" / "화면ID" / "화면경로", Noto Sans KR Bold 9pt, #000000, x=16 |
| **메타 값** | Noto Sans KR Regular 9pt, #000000, x=120 |
| **구분선** | Line, x=16, w=403, Stroke #DDDDDD |
| **기능 상세 제목** | Noto Sans KR Bold 9pt, "기능 상세 (이전 프레임 ①~N과 동일 항목 생략)" |
| **불렛포인트** | Noto Sans KR Regular 9pt, 번호 + 기능 설명, textAutoResize HEIGHT, w=403 |

구성 순서 (위→아래):
1. 타이틀 바 ("화면 명세")
2. 화면명 라벨 + 값 (y=48)
3. 화면ID 라벨 + 값 (y=69)
4. 화면경로 라벨 + 값 (y=90)
5. 구분선 (y=115)
6. 기능 상세 섹션 제목 (y=127)
7. 불렛포인트 텍스트 블록 (y=148~)

상세 프레임의 경우: "※ 이전 프레임 ①~N과 동일 항목 생략" 문구를 불렛포인트 최상단에 표기

### 컴포넌트 테이블 프레임 (752 x 가변)

| 요소 | 스타일 |
|------|--------|

### 컴포넌트 테이블 "설명" 컬럼 작성 원칙

> 핵심 원칙: 컴포넌트가 **무엇을 하는지(기능/동작)**가 아니라, **어떤 UI 컴포넌트인지(유형/성격)**를 기술한다.
> 기능/동작 설명은 화면 명세의 불렛포인트에서 담당하므로, 컴포넌트 테이블과 역할이 중복되지 않도록 한다.

### 위치 규칙

- 화면 명세 프레임: 와이어프레임 프레임 오른쪽으로 **간격 100px** 이상, 와이어프레임 상단과 y 정렬
- 컴포넌트 테이블 프레임: 화면 명세 프레임 바로 아래 **간격 30px**, 또는 화면 명세 프레임과 같은 x에서 아래로 배치
- 상세 프레임의 화면 명세/컴포넌트 테이블도 동일 패턴으로 해당 와이어프레임 옆에 배치

### 네이밍 규칙

- 화면 명세: `[화면명] - 화면 명세` (예: "검도 작업 결과 상세 (상세 - 하단 테이블 펼침) - 화면 명세")
- 컴포넌트 테이블: `[화면ID]_컴포넌트_테이블` (예: "검도_결과_상세_하단테이블_컴포넌트_테이블")

## GNB 복제 규칙

- 새 화면 프레임 작성 시, 기존 GNB 컴포넌트를 인스턴스로 복제하여 동일하게 배치
- GNB 구성: 배경(#F2F2F2) + Logo(Rounded Rectangle #222) + 메뉴 텍스트 + 현재 페이지 Bold + Bell/User 아이콘
- 와이어프레임 프레임은 GNB부터 시작 (헤더테이블/구분선 없음) — 화면 메타정보는 화면 명세 프레임에 기재

## 상세 화면 주석 번호 연속 규칙

- 예: 기본 ①~⑬ → 상세 ⑭~ → 상세2 ⑳~
- 이전 프레임 마지막 번호 다음부터 연속 배정

## 프레임 삭제 절대 금지

## 새 화면 작성 전 위치 확인

- 새로운 화면설계서 프레임 작성 전, 반드시 사용자에게 어느 프레임 뒤에 삽입할지 먼저 질문한다
- 예: "몇 번 프레임 뒤에 추가할까요?"
- 다른 기존 프레임은 구조 참고용으로만 활용하고, 실제 작성할 프레임 위치를 사용자에게 먼저 확인한 후 작업을 시작한다
- 위치 확인 전에는 프레임 복제나 편집을 시작하지 않는다
