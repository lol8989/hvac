# LG 디자인 시스템 v1 — 관리자 영역

> 작성일 2026-07-10 · 지시: 주인님 (LG 시그니처 색상 기반 디자인 시스템, 관리자 우선 적용)
> 상태: **관리자 영역 적용 완료**. 생성·검도는 무채색 유지.

## 1. 결정 사항

| 항목 | 결정 | 근거 |
|------|------|------|
| 유채색 금지 규칙 | **POC 앱은 예외** (주인님 확정) | 화면설계서(Figma) 무채색 규칙은 그대로 유지 |
| 액센트 강도 | **절제형** — Primary 액션·활성 네비·포커스·선택 행에만 | 1,206행 테이블이 주인공. 붉은색을 넓게 깔면 상태 뱃지·경고와 충돌 |
| 상태 뱃지 | **의미색 도입** (게시/작성중/단종) | 전 상태 1,206행을 훑을 때 상태가 한눈에 잡혀야 함 |
| 테마 | **단일 라이트 테마** | 종일 켜 두는 업무 콘솔이고, 흰 바탕 도면·스펙시트와 나란히 본다 |
| 적용 범위 | **관리자(`.adm-root`)만** | 생성·검도는 도면 콘텐츠가 주인공 → 무채색 유지 |

## 2. 색상

### 브랜드

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--lg-red` | `#A50034` | LG Active Red. Primary 버튼·활성 메뉴·포커스 링·선택 레일 |
| `--lg-red-deep` | `#7E0027` | hover / press |
| `--lg-red-wash` | `#FBF1F4` | 선택 행 · 활성 메뉴 배경 |
| `--lg-red-line` | `#E8C9D3` | 레드 계열 경계선 |
| `--lg-black` | `#1A1417` | GNB 그라운드 |

`--lg-black`은 순수 검정이 아니라 붉은 기가 미세하게 도는 근사 검정이다. 순검정 위에서 Active Red는 차갑게 뜬다.

### 뉴트럴 (액센트 쪽으로 편향된 웜그레이)

`--n-0 #FFFFFF` · `--n-50 #F8F6F7` · `--n-100 #F2EEF0` · `--n-150 #ECE7E9` · `--n-200 #E2DCDF` · `--n-300 #D3CBCF` · `--n-500 #7C7378` · `--n-700 #4A4348` · `--n-800 #373135` · `--n-900 #1B171A`

순수 회색(`#F5F5F5`, `#888`)을 쓰지 않는다. 브랜드 레드 옆에서 차갑게 튀어 "고르지 않은 색"으로 읽힌다.

### 의미색 (채도를 낮춰 브랜드 레드와 구분)

| 상태 | 전경 | 배경 | 경계선 |
|------|------|------|--------|
| 게시 (PUBLISHED) | `--ok #1B6B4A` | `--ok-bg #E8F1EC` | `--ok-line #C3DDCF` |
| 작성중 (DRAFT) | `--idle #5C545A` | `--idle-bg #F2EEF0` | `--idle-line #DDD5D9` |
| 단종 (ARCHIVED) | `--stop #7A4A52` | `--stop-bg #F3EBEC` | `--stop-line #E0CDD1` |

뱃지는 **색 + 점(dot)** 두 채널로 상태를 인코딩한다(`::before` 원형). 색각 이상에서도 형태로 구분된다.

폼 오류는 브랜드 레드가 아니라 `--stop` 계열을 쓴다. 오류와 Primary 액션이 같은 색이면 혼동된다.

## 3. 타이포

| 역할 | 폰트 | 비고 |
|------|------|------|
| UI | `--font-ui` Noto Sans KR | 외부 폰트 CDN을 로드하지 않는다(오프라인 무음 폴백 방지) |
| 데이터 | `--font-data` Consolas / D2Coding | 모델명 등 고정폭 |

스케일: `--fs-xs 12` · `--fs-sm 13` · `--fs-md 14` · `--fs-lg 15` · `--fs-xl 17` · `--fs-2xl 20`

- 본문·데이터·버튼·입력은 **14px 하한**(기존 정책 유지). 뱃지·힌트만 12px.
- 숫자가 열을 이루는 곳은 전부 `font-variant-numeric: tabular-nums` (HP·용량·건수·페이지·날짜).
- 소형 라벨(테이블 헤더·사이드 헤더)은 `--tracking-label 0.04em`.

## 4. 레이아웃

```
┌─ GNB (LG Black, 활성 메뉴에 Red 밑줄) ───────────────┐
├──────────┬───────────────────────────────────────────┤
│ 사이드    │ 서브 헤더 (타이틀 좌측 정렬 + 요약 pill)    │
│ 208px    │ 툴바 (필터·검색·표시건수·액션)              │
│ 활성=Red │ 일괄 작업 바                                │
│ 레일+wash│ ┌─ 테이블 (여기만 스크롤, sticky 헤더) ──┐  │
│          │ └────────────────────────────────────────┘  │
│          │ 페이저 (하단 고정)                          │
└──────────┴───────────────────────────────────────────┘
```

**스크롤 주체는 테이블이다.** 페이지 전체가 스크롤되면 툴바·일괄 바·페이저가 화면 밖으로 밀린다. 100건 표시에서도 액션은 항상 손 닿는 곳에 있다.

⚠️ 두 가지 함정을 겪고 고쳤다.
1. `overflow-x: auto`는 세로 축도 스크롤 컨테이너로 만든다 → `thead`의 `position: sticky; top: 0`이 뷰포트가 아니라 그 컨테이너 기준으로 붙는다. 스크롤 주체를 테이블 래퍼로 명시해야 한다.
2. 모서리 라운딩을 `overflow: hidden`으로 깎으면 **테이블 자체가 스크롤 컨테이너가 되어 sticky 헤더가 멎는다**. 모서리 셀에 `border-radius`를 직접 준다.

## 5. 컴포넌트 규칙

- **Primary는 화면당 하나의 결(結)** — 등록/게시처럼 그 화면의 주 행동에만 Red. 나머지는 뉴트럴 보더 버튼.
- **선택 행**: `--lg-red-wash` 배경 + 좌측 3px Red 레일(`box-shadow: inset`).
- **일괄 바 건수**: 선택이 0건이면 강조하지 않는다(`has-sel` 클래스). 0을 브랜드 색으로 외치면 거짓 신호다.
- **포커스**: `.adm-root :focus-visible`에 2px Red 아웃라인 + 오프셋. 키보드 사용자에게 항상 보인다.
- **모션**: 0.12s 색 전이만. `prefers-reduced-motion: reduce`에서 전부 끈다.
