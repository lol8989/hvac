## 1. 요구사항 (원문)

- 하단 플로팅 메뉴(Figma형 도구바)에 **실 자르기** 항목을 추가하고 단축키는 **V**.
- V 모드에서는 **마우스 포인터 대신 라인이 형성된다**.
- 포인터가 라인인 상태에서 **실을 클릭하면 그 위치에서 실이 슬라이싱된다**.
- 슬라이스 모드에서 **R을 누르면 라인이 15°씩 회전**하고, 회전한 각도로 자를 수 있다.

## 3. 계획 수립 중 확정한 파생 규칙 (조사에서 드러난 것)

### 3.1 자식 면적은 "부모 면적 × 넓이 비율"이다

목업 도면에는 **전역 px→㎡ 축척이 없다**. `AC_001`(31.89㎡)과 `AC_004`(42.0㎡)는 픽셀 박스가 250×150으로 **같다**(`src/data.ts:60,63`). 실마다 `scale = √(area / (wPx·hPx))`를 따로 잡기 때문이다(`data.ts:32-37`).

따라서 자른 조각의 면적을 픽셀 기하에서 직접 유도할 수 없다. 규칙:

```
areaM2(자식) = areaM2(부모) × ( 폴리곤넓이(자식) / 폴리곤넓이(부모) )
```

이 규칙은 **면적 합 보존**(`area(A) + area(B) === area(부모)`)을 자동으로 만족하고, 실도면(mmPerUnit 있음)에서도 부모 면적이 기하와 정합하면 동일한 값을 낸다. 부모의 축척을 그대로 물려받는 셈이라 `data.ts`의 기존 관례와도 어긋나지 않는다.

### 3.2 자식 단변/장변은 OBB에서 새로 뽑는다

`Room.withArea`는 `shortSideM/longSideM`을 **그대로 물려준다**(`Room.ts:107-109`). 면적만 바꾸면 실내기 타입 결정(`indoorTypeFor`의 4.0m/3.0m 폭 경계)이 **옛 형상 기준**으로 나온다. 따라서 자식은 반드시 `Room.create`로 새로 만들고, 단변/장변은 자식 폴리곤의 **최소면적 회전 바운딩박스(OBB)** × 부모 축척으로 계산한다.

### 3.3 조각에 심볼이 없으면 그 실은 "미배치"다

D3(심볼 위치로 분배)의 자연스러운 귀결. 한쪽 조각에 심볼이 0개면 그 실의 `Placement`를 만들지 않는다(`quantity >= 1` 불변식을 우회하지 않는다). 다음 단계로 넘어갈 때 기존 가드 `ROOMS_WITHOUT_INDOOR`가 "실내기가 없는 실"로 잡아 준다 — 새 예외 경로를 만들 필요가 없다.

### 3.4 너무 얇은 조각은 자르지 않는다

절단선이 실의 꼭짓점을 스치면 면적 0에 수렴하는 조각이 생기고, `Room.create`의 `assertPositiveFinite`(`Room.ts:53-55`)나 `layoutPositions`의 폭 가드가 throw한다. **자르기 전에 거부**한다: 조각 넓이가 부모의 **2% 미만**이면 자르지 않고 토스트로 알린다("너무 얇게 잘려 실을 만들 수 없습니다").

**Polygon은 단위를 모른다.** 넓이는 "폴리곤 넓이"(px²든 mm²든)일 뿐이고, ㎡ 환산은 `sliceRoom`이 부모 면적 비율로 한다(§3.1). 도메인이 SVG 픽셀을 아는 일은 없다.

## 5. 구조적 선결 과제 — 실 기하의 SSOT가 없다

지금 실의 좌표 소유자는 **모듈 상수** `ROOMS`(`src/data.ts:59-66`)다. `viewRooms`가 `const r = ROOMS[id]`로 룩업한다(`App.tsx:164`).

→ **`ROOMS`에 없는 새 id(=슬라이스로 태어난 실)는 좌표를 가질 수 없다.** 스프레드가 `undefined`를 조용히 무시하므로 x/y/w/h가 사라지고, 런타임 에러 없이 뷰어에서 실이 사라지거나 `layoutPositions`가 나중에 throw한다. `noUncheckedIndexedAccess`가 꺼져 있어 **타입도 못 잡는다.**

그래서 슬라이싱보다 먼저:

```ts
const [roomGeom, setRoomGeom] = useState<Record<string, RoomGeom>>({})  // 실 기하 SSOT
```

`ROOMS`는 "AI 검출기의 목업 출력"으로 격하한다 — `detectRooms()`가 이걸 읽어 `domainRooms`와 `roomGeom`을 함께 시딩한다. (P0)

## 6. 구현 순서 (TDD — Red 후보 명시)

### P0. 실 기하 SSOT 승격 — 슬라이싱 없이 선행
- **Red**: `ROOMS`에 없는 id의 실이 `domainRooms`에 있어도 뷰어에 그려지고 `layoutFor`가 throw하지 않는다.
- App: `roomGeom` state 신설 → `viewRooms`(`App.tsx:160-169`)·`layoutFor`(`:213`)·`detectRooms`(`:731`)가 이걸 읽는다.

### P1. `domain/shared/Polygon.ts`
- **Red**: 직사각형 폴리곤의 shoelace 면적 = w×h / 경계 위의 점은 내부로 친다(`pointInZone` 기존 계약 유지) / 정사각형 OBB는 short === long / 45° 기울어진 사각형의 OBB가 축정렬 bbox보다 작다.
- **Red**: `splitByLine` — ① 사각형을 수직으로 자르면 사각형 2개 ② **두 조각 넓이 합 = 원본** ③ 선이 폴리곤을 안 지나면 조각 1개(원본) ④ 15°/30°/45° 사선에서 정점 수가 3~5개로 나온다.

### P2. `domain/generation/sliceRoom.ts`
- **Red**: 자식 면적 합 = 부모 면적 / 자식 `floor`·`usage`·`facility` = 부모 상속 / 자식 id에 `#`가 없다(`AC_001-1`) / 자식 단변·장변이 **자식 OBB에서 나온다**(부모 값을 물려받지 않는다) / 부모가 4WAY였어도 좁게 자른 자식은 `indoorTypeFor`가 2WAY를 낸다 / 부모의 단위부하 user 오버라이드는 자식에 승계된다 / 조각이 부모의 2% 미만이면 `TooThinSliceError`.

### P3. `layoutPositions` 폴리곤 일반화
- **Red**: 반환된 모든 좌표가 **폴리곤 내부**다(현행 rect 버전 → point-in-polygon) / 삼각형 실에 4대를 요청해도 4좌표 전부 내부 / 좌표 중복 없음.
- 이걸 안 고치면 사다리꼴 모서리 셀이 벽 밖에 찍히고, `zoneOfPoint`가 그 심볼을 **옆 실 소속**으로 오분류한다(위치가 소속을 이긴다 — `geometry.ts:54-56`) → 선정표 대수가 옆 실로 넘어간다.

### P4. 뷰어 기하 폴리곤 전환 (`geometry.ts` + `ZoneRect.tsx`)
- `ZoneBox {x,y,w,h}` → `{points}`. **여기를 바꾸면 나머지 사각형 가정이 전부 컴파일 에러로 드러난다(안전망).**
- `zoneAreaM2`는 shoelace로. ⚠️ **타입이 못 잡는 유일한 축** — 그대로 두면 조용히 bbox 면적(과대)이 나와 부하→대수→조합비까지 오염된다. 테스트로 고정한다.
- `rectsIntersect`(마퀴) → 폴리곤 교차. `pointInZone` → 폴리곤 포함.
- `ZoneRect`: `<polygon>` + 라벨 앵커를 centroid로(삼각형에서 좌상단 인셋은 실 밖으로 나간다).
- 정점 리사이즈는 **1차 범위 밖**(사각형 코너 리사이즈는 사각형 실에 한해 유지, 폴리곤 실은 리사이즈 비활성).

## 7. 건드리는 불변식

1. `Placement`: `positions.length === quantity`, `quantity >= 1`
2. `Room`: `areaM2·shortSideM·longSideM > 0`, `short = min, long = max`
3. `AssignmentPlan`: 한 실의 모든 대수는 같은 그룹(실 응집) → **부모 유닛을 재라벨하지 말고 전부 없앤 뒤 자식 유닛을 새로 만든다**
4. 유닛 id 규약 `` `${roomId}#${n}` `` → 실 id에 `#` 금지
5. 실외기 버킷 = 층 × 계열 → 자식은 부모의 **층을 반드시 상속**
6. **(신설)** 면적 합 보존: `area(A) + area(B) === area(부모)`
7. **(신설)** 심볼 좌표는 항상 실 폴리곤 내부
