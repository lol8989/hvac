## 1. 방향 (왜 바꿨나)

기존 조합비는 "검증용 고정 상수(0.5~1.3) 체크"였다. 그러나 실무 산출물인 **장비선정표 엑셀**을 분석한 결과, 조합비는 파이프라인 맨 끝에 **결과로 찍히는 값**이고 진짜 도메인은 그 앞의 수치 체인이다:

```
층별·실명(유저) → 면적(검출) → 단위부하(kcal/h·㎡, 용도별) → ×1.163 W 변환
→ 필요부하량 = 단위부하 × 면적 → 실내기 모델+대수(AI 추천) → 실외기 그룹핑 → 조합비
```

**실검출·AI 실내기 배치·실외기 배치는 결국 이 양식을 뽑아내기 위한 행위다.**

### 엑셀 양식 분석 요약
- 냉난방 계열 탭(Multi V Super/GHP, DOAS, 하이브리드, Mini/Multi V S, All in 1, 중대형)은 **동일 컬럼 골격** 공유. eco-V(환기)만 별도 양식(체적·재실인원·환기횟수) — POC 범위 밖.
- 행=실. 컬럼: 층별 | 실명 | 면적㎡ | 단위부하(kcal/h·㎡ 냉/난) | 단위부하(W) | 필요부하량(W) | 실내기 장비번호(40C/110T)+모델명+냉난방용량+**대수**+총용량 | 실외기 장비번호(**HP**)+모델명+용량+대수+**조합비** | 비고. 층 합계 행 + 하단 BOM(모델별 대수, HP 합계).
- 조합비 산식 검산: 그룹 총냉방W ÷ 실외기 냉방W (지하1층 18000/23300=0.7725 ✓).
- **범위 상충 증거**: NOTE 3 "조합비 100% 이내" ↔ 같은 시트 실데이터 1.106(GHP), DOAS 탭 0.32 — 범위는 제품군별로 다르다.

## 4. 선정표 '새 창' 아키텍처 (결정 #2)

- 메인 창(생성 작업) 실외기 조합·산출물 단계의 **[⧉ 선정표 확인]** → `?view=selection` 팝업(1480×860, 이름 있는 창이라 재사용).
- 동기화: `BroadcastChannel('poc-selection')` — 새 창이 `hello` → 메인이 스냅샷 응답. 새 창의 셀 편집은 `edit` 커맨드로 메인 상태(SSOT)에 반영 → 메인이 갱신 스냅샷 재방송(단방향 데이터 흐름). 프로토콜: `src/presentation/generation/selectionSync.ts`.
- 편집 셀: 실명·단위부하(냉/난)·모델·대수·그룹. 자동 계산: W변환·필요부하·총용량·조합비·층합계·BOM. 셀별 `AI`/`수정` 뱃지 + ↺ 초기화.
