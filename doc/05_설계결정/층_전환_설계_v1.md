## 1. 배경 · 문제

실무 현장은 **다층이 기본**(지하~지상 수십 층, 실 수백 개. 실측 008.dxf = 검출 588실). 전 층 실을
한 캔버스에 다 올리면 도면이 "뭉텅이"가 되어 읽을 수 없다.

**개발자 버전의 답(반례):** 좌측에 **상주 층별 방 리스트**(`1층 164개·5811㎡`, `방 550 / 6.0㎡` … 수백 줄).
- 이름이 `방 550, 551, 552…`로 저정보 → 스크롤해도 못 찾는다. 캔버스에 이미 있는 걸 정보 없이 복제.
- **층을 사이드 리스트 필터로만 취급** — 정작 캔버스는 전 층을 다 쏟아 읽기 불가. 방향이 거꾸로.
- 260px 상주 크롬이 도면 영역을 잠식 → **도면 중심 원칙 위반.**

**원 기획(정본):** `.viewer{flex:1}` 도면이 주인공 + `.rpanel` **우측 패널 하나** + `.room{cursor:pointer}`
캔버스 클릭 선택. **좌측 패널 없음.** 층 전환은 이 도면 중심을 지키면서 다층을 푸는 정공법이다.

**핵심 결정:** 층은 **사이드 목록이 아니라 그림을 가르는 축**이다 → **캔버스가 한 번에 한 층**을 보여준다.

---

## 2. 원칙

1. **도면 중심 유지** — 좌측 상주 패널 신설 금지. 층 전환은 뷰어 상단의 가벼운 탭/셀렉터.
2. **층은 이미 도메인 축** — `Room.floor` 존재, 실외기 버킷 = 층 × 계열(한 실외기가 층을 안 넘음,
   `selectOutdoorPlan`이 이미 `(roomId)=>floor`로 버킷팅). 뷰어는 이 축을 화면에 반영할 뿐이다.
3. **선정표가 다층 색인을 겸한다** — `SelectionGrid`는 이미 층별 그룹핑(`section.floor 합계`, floorcell).
   "전 층을 표로 훑기"는 여기서 된다 → 좌측 리스트로 중복 구현하지 않는다. 뷰어는 **단층 편집** 전용.

---

## 3. 데이터 모델

- `Room.floor: string` — 검출 워커가 부여(POC는 하드코딩 `'지상1층'`). 뷰어는 이 값만 신뢰한다.
- 실 형상 `geom`, 실내기 `placements`, 실외기 좌표 `outdoorPositions`는 **실 id로 층에 귀속**(별도 층 필드 불필요).
- **하나의 DXF에 전 층이 공간 배치**된다(008.dxf, 588실). 즉 층은 한 도면 안의 **논리 분할**이다
  (층별 별도 파일이 아니다).

---

## 4. 뷰어 동작

- **뷰 상태** `activeFloor`(기본: 정렬상 첫 층). **World(되돌리기 단위) 밖**에 둔다 — 층 전환은 편집이
  아니라 뷰 이동이므로 Ctrl+Z가 되돌리면 안 된다(`panelOpen`/`panelW`와 같은 성격).
- **렌더 필터**: `geom`·`placements` 중 `activeFloor` 실만 그린다(그 외 숨김. 겹침 레이아웃 대비 필수).
- **프레이밍**: `activeFloor` 실들의 bbox로 자동 fit(pan/zoom). 층이 나란히(side-by-side) 배치면
  이 fit만으로 다른 층이 자연 격리된다.
- **선택·편집 스코프**: 자르기(V)·병합(M)·실내기 배치·실외기 배치는 모두 `activeFloor` 실만 대상.
  마퀴·Del·자르기가 다른 층 객체를 건드리지 않는다(적대적 QA 항목).

---

## 5. 층 목록 · 정렬

- 층 목록 = distinct `Room.floor`. 정렬 규칙이 필요하다(문자열 정렬은 `지하10층 < 지하2층`으로 깨진다).
- **domain util `floorOrder(floor: string): number`**
  - `지하N층` / `BN` → `-N` · `지상N층` / `N층` / `NF` → `+N` · `옥탑` / `RF` / `PH` → 매우 큰 수
  - 파싱 실패 → 뒤로 밀되(큰 수), 원문 보존해 탭에 그대로 표기.
- 각 층 탭에 실 수·합계 면적 표기 가능(`SelectionGrid`의 층 집계 로직 재사용 — 신규 계산 아님).

---

## 6. UI

- 뷰어 **상단 좌측에 층 탭**(층 ≤ N개면 탭, 많으면 드롭다운). 활성 층 강조(디자인 토큰 Red 액센트).
- 층 전환 시: 해당 층 실만 캔버스 + bbox fit + **선택 초기화**(v1 기본. 층별 선택 보존은 개선안).
- **단층 프로젝트**(모든 실 같은 층)면 탭 1개 → 현재 동작과 동일. 무해한 degrade.

---

## 7. 도메인 · 애플리케이션 영향

| 대상 | 영향 |
|------|------|
| 실외기 선정 `selectOutdoorPlan` | **변경 없음** — 이미 층 accessor로 버킷팅 |
| 선정표 · CSV · 산출 도면 | **다층 유지**(전 층). 뷰어만 단층 |
| `World` | `activeFloor`는 **넣지 않는다**(뷰 상태) |
| 실외기 이격(200/900mm) 검증 | 같은 층 안에서만 의미 — 층 스코프와 정합 |

## 8. 오픈 이슈 (구현 전 확인)

1. **층 레이아웃 = 나란히(side-by-side)인가 겹침(stacked)인가?** 검출 워커 출력에 달림.
   - 렌더 필터(by-floor)는 양쪽 다 안전. bbox-fit은 나란히를 전제 → 겹침이면 fit이 전 층을 다 잡는다.
   - v1은 **필터 우선 + bbox-fit 보조**로 양쪽 안전하게 간다.
2. **층 문자열 표준**(`지상1층` vs `1F` vs `1층`) — 검출 워커와 합의 필요. `floorOrder`가 다형 파싱으로 흡수하되 표준 1개 권장.
3. **층 전환 시 선택 정책** — v1: 초기화. 개선: 층별 선택 보존.
