# 장비마스터 운영정책서

> 대상: 장비마스터(Equipment Master) 관리자
> 버전: v1.0 (2026-07-09) — 장비 등록 기능(P2-S2 · P3) 구현 완료 시점 기준
> 근거 커밋: `15fc1e0`(쓰기 포트·등록/수정/게시) · `35c7d38`(단가 제거) · `2ab7a12`(스펙시트 업로드)

---

## 1. 목적과 원칙

장비마스터는 장비 모델·스펙의 **단일 진실 공급원(SSOT)** 이다. 등록의 목적은 등록 그 자체가 아니라 **장비선정표와 장비일람표를 채우는 것**이다. 따라서 어떤 필드를 정규화할지는 항상 산출물이 소비하는 컬럼에서 역으로 결정한다.

| 원칙 | 내용 |
|------|------|
| 등록만 담당 | 마스터는 모델·스펙을 등록·게시한다. 호환·조합 판단(조합비·계열 매칭)은 유저(생성) 단이 마스터를 참조해 수행한다. |
| 게시 게이트 | 검도·생성은 **PUBLISHED 데이터만** 조회한다. 등록만으로는 산출물에 반영되지 않는다. |
| 역방향 의존 금지 | 마스터는 생성·검도를 알지 못한다(Customer/Supplier, 마스터가 상류). |
| 원본 우선 | LG전자 스펙시트 원본(xlsx)을 그대로 올린다. 별도 양식으로 옮겨 적지 않는다. |

---

## 2. 분류 체계

4단 계층으로 관리한다. 등록 시 관리자는 **시리즈**를 선택하고, 그 위 계층(대분류·중분류)은 시리즈에 종속되어 자동 결정된다.

```
대분류(역할)   실외기 / 실내기
  └ 중분류(제품군·계열)   냉난방 절환형 · 냉방전용 · GHP / 4WAY 카세트 · 덕트
      └ 시리즈   Multi V Super 절환형, GHP Super, Multi V 실내기 4WAY …
          └ 모델   RPUW281X9P, GPUW280C2S, RNW0401C2S  ← 마스터 레코드
```

계열(EnergySource: EHP / GHP / AWHP …)은 중분류에 귀속된다. 계열은 생성 단의 **호환 판정 기준**이므로, 시리즈를 잘못 고르면 실외기–실내기 조합이 어긋난다. 업로드 시 시리즈 선택은 되돌리기 어려운 결정으로 취급한다.

## 3. 게시 상태와 상태 전이

### 3.1 세 가지 상태

| 상태 | 표기 | 의미 | 생성·검도 노출 |
|------|------|------|----------------|
| `DRAFT` | 작성중 | 등록되었으나 미검증 | ✗ |
| `PUBLISHED` | 게시 | 검증 완료, 사용 가능 | **○** |
| `ARCHIVED` | 보관 | 단종·폐기 | ✗ |

### 3.2 허용 전이 (도메인 불변식으로 강제)

```
  DRAFT ──게시──▶ PUBLISHED
    │                  │
   폐기              보관
    │                  ▼
    └────────────▶ ARCHIVED
                       │
                     재게시
                       ▼
                  PUBLISHED
```

- **금지**: `PUBLISHED → DRAFT`(게시 취소), `ARCHIVED → DRAFT`(보관 해제), 같은 상태로의 전이
- **금지 이유**: 게시본을 되돌리면 그 모델을 이미 참조한 생성·검도 결과가 근거를 잃는다. 되살릴 때는 **재게시**(`ARCHIVED → PUBLISHED`)를 쓴다.
- 위반 시 `INVALID_TRANSITION` 예외 → 저장소가 롤백하고 UI가 사유를 안내한다.

### 3.3 게시본 잠금

**스펙 수정은 `DRAFT` 상태에서만 가능하다.** 게시·보관본의 수정 버튼은 비활성이며, 우회 호출 시 `SPEC_LOCKED` 예외로 거부된다.

> 게시된 모델의 스펙을 고쳐야 한다면: ① 보관(ARCHIVED) 처리 후 새 모델로 등록하거나, ② 오탈자 수준이면 보관 → 재등록 절차를 따른다. 발밑에서 SSOT가 바뀌는 상황을 만들지 않는다.

### 3.4 게시 절차 (권장)

1. 업로드 또는 수기 등록 → 자동으로 `DRAFT`
2. 목록에서 상태 필터 `작성중`으로 좁혀 HP·냉난방 용량·최대 연결 실내기 수를 눈으로 확인
3. 이상 없으면 행의 `게시` 클릭 → `PUBLISHED`, `published_at` 기록
4. 잘못 올렸으면 `폐기`(DRAFT → ARCHIVED). 삭제 기능은 없다(이력 보존).

## 4. 등록 경로

### 4.1 스펙시트 업로드 (주 경로)

LG전자 스펙시트 원본 xlsx를 그대로 올린다. 파일당 여러 모델이 열 방향으로 들어 있다.

**절차**: `스펙시트 업로드` → 시리즈 선택 → 파일 선택 → **검증 요약 확인** → 업로드

| 제약 | 값 |
|------|-----|
| 형식 | `.xlsx` 만 |
| 최대 크기 | 10 MB |
| 적재 상태 | 항상 `DRAFT` |
| 트랜잭션 | 전건 일괄. 하나라도 실패하면 **아무것도 남지 않는다** |

**검증 요약(업로드 전 미리보기)** — 네 가지 수치로 제시한다.

| 항목 | 의미 | 처리 |
|------|------|------|
| 총 모델 수 | 시트에서 감지한 모델 열 수 | — |
| 정상 (등록 대상) | 불변식을 모두 통과 | 적재 |
| 오류 (스킵) | 모델명 없음 / 냉·난방 용량 모두 없음 / 실외기인데 HP 유도 불가 | 제외 |
| 중복 (스킵) | 마스터에 이미 있는 모델명, 또는 같은 파일 안의 중복 | 제외 |

모델명 비교는 **대소문자·앞뒤 공백을 무시**한다. 같은 파일을 두 번 올리면 두 번째는 전량 중복으로 스킵되므로, 재업로드는 안전하다.

### 4.2 수기 등록·수정 (보조 경로)

`＋ 제품 등록` 폼은 정규화 hot 필드(시리즈·모델명·장비번호·냉방·난방·마력·최대 연결 실내기 수)만 입력한다. **롱테일 스펙을 채울 수 없으므로 일람표 확장 컬럼이 비게 된다.** 오탈자 보정이나 예외 모델에만 쓰고, 정식 등록은 업로드를 사용한다.

### 4.3 중복 제출 방지

저장·수정·게시·업로드 버튼은 처리 중 잠기며, 연타해도 **1회만** 실행된다. (동기 잠금 + 버튼 비활성)

## 5. 스펙 정규화 규칙

### 5.1 시트 구조

```
[헤더]   A='항목'    B=∅        C='단위'|'모델명'    D~ = 모델명들
[범례]   A='대분류'  B='소분류'                        (Multi V 계열에만 존재)
[데이터] A=대분류    B=소분류   C=단위               D~ = 모델별 값
```

헤더 행 위치는 파일마다 다르다(Multi V는 1행, GHP는 공백행 뒤 2행). 파서가 자동 탐지한다.

### 5.2 정규화 필드 (products 테이블 컬럼)

| 필드 | 출처 | 규칙 |
|------|------|------|
| `model_code` | 헤더 D열~ | 전역 고유. 중복 시 등록 거부 |
| `cooling_capacity_w` | 라벨에 '냉방' + 단위 `kW`인 **첫** 행 | kW → W 정수 변환 |
| `heating_capacity_w` | 라벨에 '난방' + 단위 `kW`인 **첫** 행 | 냉방전용이면 없음(null) |
| `max_connections` | 라벨에 '실내기'와 '연결'을 함께 포함 | 정수 |
| `horsepower` | **모델명에서 유도** (§5.4) | 실외기만 |

**능력 오인 방지**: 라벨에 `소비·전력·효율·전류·연료·한랭·저온` 중 하나라도 있으면 능력 행이 아니다. (소비전력·연료 소비량·저온(-15℃) 능력도 같은 `kW` 단위를 쓰기 때문)

**정격의 정의**: 능력 행이 여러 개면(FCU의 `능력조건 A/B` 등) **시트에서 처음 나오는 행**을 정격으로 채택한다.

### 5.3 롱테일 스펙 (product_specs JSONB)

정규화하지 않은 모든 행을 `"대분류 > 소분류"` 키로 값·단위와 함께 통째 보존한다. 값이 `-` 또는 공란인 칸은 저장하지 않는다.

```json
{
  "전원 > Case 1":                    { "value": "380, 3상(4선), 60", "unit": "V, Phase, Hz" },
  "냉매 연결 배관경 > 액관":            { "value": "Φ19.05 (3/4)",      "unit": "mm(inch)" },
  "연결전선 > 전원선(H07RN-F, 접지포함)": { "value": "16.0 × 5C",         "unit": "mm² × cores" },
  "전기특성치 > 차단기(ELCB)":          { "value": "75",                "unit": "A" },
  "냉매 > 종류":                       { "value": "R410A",             "unit": null }
}
```

일람표의 전원·배관경·전선·차단기·냉매·소음·중량·치수 컬럼은 여기서 조인한다.

### 5.4 실외기 마력(HP) 유도 규칙 — 중요

**스펙시트에는 마력 항목이 없다.** (50개 파일 전수 조사: 냉장·냉동 CDU 3건 예외) 대신 모델명이 HP를 인코딩한다.

> **규칙**: 접두 알파벳(하이픈 허용) 뒤 **첫 두 자리 숫자**가 HP다.

| 모델명 | HP | 비고 |
|--------|----|----|
| `RPUW281X9P` | 28 | |
| `RPUB081X9E` | 8 | 선행 0 |
| `RP-B261X9E` | 26 | 하이픈 접두 |
| `GPUW280C2S` | 28 | GHP |
| `GP-W560C2S` | 56 | 280×2 조합 모델 |
| `RPUW12BX9M` | 12 | 세대 문자가 3번째 자리 |

**kW를 HP로 나누는 변환식은 쓰지 않는다.** 계열마다 kW/HP 비가 달라 성립하지 않는다.

| 계열 | 예시 | 냉방 | ÷2.8 | 실제 HP |
|------|------|------|------|---------|
| 고급형 | RPUW281X9P | 78.4 kW | 28.00 | 28 |
| 동시형 | RPUB081X9E | 23.3 kW | **8.32** | 8 |
| GHP | GPUW280C2S | 82 kW | **29.29** | 28 |

유도 실패(샤시명 `UXB` 등)나 범위(1~99) 밖이면 **오류 행으로 스킵**한다. 실내기는 HP를 요구하지 않는다.

## 6. 산출물 연결

| 산출물 | 마스터가 공급하는 것 |
|--------|---------------------|
| 장비선정표 | 실내기 `장비번호·모델·유형·냉난방 용량` / 실외기 `HP·모델·냉방kW·난방kW·최대 연결 수` |
| 장비일람표 | 위 + `product_specs`의 롱테일 컬럼(전원·배관경·전선·차단기·냉매 …) |

생성·검도는 **PUBLISHED 스냅샷만** 읽는다. 부팅 시 1회 materialize하므로, **게시 직후 생성 화면에 반영하려면 새로고침**이 필요하다.

> **단가**: 현재 관리 UI에서 단가 편집·표시 경로는 제거되어 있다(2026-07-09 결정). 스펙시트에 단가가 없기 때문이다. 스키마(`product_prices`)와 도메인 `Price` VO는 존치하므로, 단가 원천이 확정되면 UI만 다시 붙이면 된다.

## 7. 예외·장애 대응

| 상황 | 시스템 동작 | 관리자 조치 |
|------|-------------|-------------|
| 업로드 중 모델명 충돌 | 전체 롤백, 아무것도 적재되지 않음 | 검증 요약의 중복 건수를 확인하고 재시도 |
| "스펙시트에서 모델을 찾지 못했습니다" | 적재 안 함 | LG 스펙시트 **원본**인지 확인(가공본·복사본은 헤더가 깨질 수 있음) |
| "엑셀 파일을 읽지 못했습니다" | 적재 안 함 | 파일 손상 여부 확인 |
| 마력 유도 불가 오류 다수 | 해당 행만 스킵 | 실내기 시트를 실외기 시리즈로 올리지 않았는지 확인 |
| 저장소(SQLite) 초기화 실패 | 관리 페이지 진입 불가 안내 | 새로고침, 브라우저 저장소 설정 확인 |

**데이터 삭제 기능은 제공하지 않는다.** 잘못 등록한 모델은 `폐기`(ARCHIVED)로 처리한다.

## 8. 미결 사항 (확정 필요)

| # | 항목 | 현재 상태 |
|---|------|-----------|
| 1 | 조합비 전역 기본 상한 | 선정표 주석은 100%(1.0), 현 POC 코드는 1.3. 관리 UI 설정값으로 해소 예정(P2-S3) |
| 2 | 제품군별 조합비 override 실수치 | 구조만 마련(`comboMin`/`comboMax`), 값 미확정 |
| 3 | 시리즈 마스터 확장 | 현재 시드 5종. 50개 스펙시트를 모두 적재하려면 시리즈·중분류 코드 체계 확정 필요 |
| 4 | 단가 원천 | 스펙시트에 없음. 장비선정표 엑셀과 조인할지 미정 |
| 5 | 실내기 장비번호(`40C` 등) 자동 부여 | 현재 업로드 시 채우지 않음(수기 입력) |
| 6 | 일람표 세부 컬럼 매핑 | `product_specs` 키 → 일람표 컬럼 대응표 미작성(P4) |

## 9. 용어

| 용어 | 코드 | 의미 |
|------|------|------|
| 실외기 | ODU / OutdoorUnit | 냉매를 실내기에 공급하는 실외 장비 |
| 실내기 | IDU / IndoorUnit | 방에 설치되는 냉난방 토출 장비 |
| 계열 | EnergySource | EHP / GHP / AWHP · 호환의 기준 |
| 조합비 | ComboRatio | Σ(연결 실내기 냉방용량) ÷ 실외기 용량 |
| 게시 게이트 | DRAFT / PUBLISHED / ARCHIVED | 외부 노출 통제 |
| 롱테일 스펙 | product_specs | 정규화하지 않은 제품군 고유 항목(JSONB) |
