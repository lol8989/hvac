생성(Generation) 파이프라인 — 화면 핸드오프

도면검도시스템 POC · 유저 화면 5단계 정적 목업 (디자이너 전달용)

기준: 현재 POC UI 재현 · 무채색(도면 콘텐츠 중심) · 각 카드를 클릭하면 전체 화면을 봅니다.

1 실 편집→ 2 실내기 배치→ 3 실외기 선정·조합→ 4 실외기 배치→ 5 산출물 생성
1
STEP · edit

실 편집

실은 이미 검출·로드된 상태. 자르기(V)·병합(M)·리사이즈로 실 형상을 다듬는다. 시설군은 프로젝트 설정.
2
STEP · place

실내기 배치

실별 부하로 타입·대수·모델 자동 선정 후 심볼 배치. 심볼 1개 = 실내기 1대 = 선정표 대수 1.
3
STEP · combine

실외기 선정·조합

실내기 정격 총용량으로 실외기 선정. 하단 도킹 패널에서 드래그로 배정/교체/분할. 선정표·일람표는 새 창.
4
STEP · outdoor

실외기 배치

선정된 실외기 심벌을 건물 외부에 배치. 이 좌표가 산출 도면에 실린다. 이격거리 검증.
5
STEP · output

산출물 생성

장비선정표(CSV)·장비일람표(xlsx)·도면(SVG) 생성·다운로드. 화면에서 놓은 좌표를 그대로 싣는다.
디자이너 참고
· 이 파일들은 현재 POC 화면을 정적으로 재현한 레이아웃/구조 기준선입니다. 인터랙션(드래그·줌·되돌리기 등)은 실제 앱에서 동작합니다.
· 생성 화면은 도면 콘텐츠가 주인공이라 무채색을 유지합니다(LG Red 액센트는 관리자 영역 전용).
· 각 화면 하단의 검은 캡션 바는 설명용이며 실제 화면에는 없습니다.
· 공통 스타일은 style.css 한 곳에 있습니다. 폰트: Noto Sans KR / 데이터: Consolas.
· 1단계는 기존 "실 검출"에서 "실 편집"으로 변경 — 진입 시 실이 이미 로드된 상태이며 별도 검출 실행 버튼이 없습니다.
